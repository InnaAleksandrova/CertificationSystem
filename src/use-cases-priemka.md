## UC 2.1. Просмотр списка заявок

|Краткое описание сценария| Просмотр списка всех заявок с возможностью фильтрации и сортировки, а также дальнейшего перехода к деталям конкретной заявки|
|---|---|
|**Действующие лица**|Административный персонал|
|**Результат**| 1. Пользователь просматривает список заявок <br> 2. Пользователь может перейти к деталям заявки|
|**Предусловие**| Пользователь авторизован в системе|
|**Основной сценарий**| <ol><li>Пользователь переходит к функциональному блоку Приемка</li><li> Система отбирает заявки для отображения в соответствии с настройками по умолчанию и отображает список заявок с их основными свойствами<li>Пользователь изменяет фильтр/порядок сортировки списка</li><li>Система перефильтрует/пересортирует список</ol>|
|**Альтернативный сценарий № 1**| 2.1. Заявки для отображения отсутствуют.<br> 2.2. Система уведомляет пользователя, что заявок для отображения нет.|
|**Детализация требований**| 1. Свойства заявки для отображения в списке: <br>- Дата регистрации <br> - Номер заявки <br> - Клиент <br> - Прогресс по заявке в формате N/M, где N - количество образцов заявки, над которыми завершена работа, M - общее число образцов заявки. <br>- Статус заявки <br><br> 2. Пользователь может фильтровать список по всем отображаемым полям, кроме прогресса|

## UC 2.2. Регистрация новой заявки

|Краткое описание сценария| Зарегистрировать новую заявку на сертификацию от клиента|
|---|---|
|**Действующие лица**| Административный персонал|
|**Результат**| Пользователь зарегистрировал новую заявку|
|**Предусловие**| Пользователь просматривает список заявок|
|Основной сценарий| <ol><li>Пользователь инициирует процедуру регистрации новой заявки.</li><li> Система отображает пользователю форму регистрации новой заявки.</li><li> Пользователь заполняет форму свойствами новой заявки и инициирует сохранение заявки </li> <li> Система создает новую заявку в статусе «Новая» и уведомляет пользователя о результате. </li>|
|**Альтернативный сценарий № 1**|3.1. Пользователь не заполнил обязательные поля и инициирует сохранение заявки <br> 3.2. Система не создает новую заявку и уведомляет пользователя о причинах|
|**Детализация требований**| Обязательные для заполнения свойства заявки: <ol><li>1. Клиент из справочника клиентов</li><li>2. Номер договора с Клиентом, к которому регистрируется заявка </ol>|

## UC 2.3 Регистрация нового образца

|Краткое описание сценария| Зарегистрировать в составе заявки образцы, которые нужно сертифицировать в рамках этой заявки. Нужно для учета образцов и фиксации требований клиента к сертификации.|
|---|---|
|**Действующие лица**| Административный персонал|
|**Результат**| Пользователь зарегистрировал новый образец в составе заявки|
|**Предусловие**| 1. Пользователь перешел к просмотру состава заявки.<br> 2. Заявка не находится в статусе «Закрыта»|
|**Основной сценарий**| <ol><li >Пользователь инициирует процедуру регистрации нового образца</li><li> Система отображает пользователю форму регистрации нового образца.</li> <li> Пользователь заполняет форму регистрации свойствами пакета и инициирует сохранение нового образца. </li><li> Система удостоверяется, что все обязательные для заполнения поля заполнены И заявка не находится в статусе "Закрыто". Система создает новый образец со статусом «В работе» в привязке к этой заявке. Система создает рабочий лист для образца и уведомляет пользователя о результате. Система открывает форму просмотра свойств зарегистрированного образца</li></ol>|
|**Альтернативный сценарий № 1**| 4.1. Система обнаруживает, что не все обязательные поля заполнены и НЕ создает новый пакет, уведомляя пользователя о результате.|
|**Альтернативный сценарий № 2**| 4.1. Система обнаруживает, что заявка находится в статусе "Закрыто" и НЕ создает новый пакет, уведомляя пользователя о результате.|
|**Детализация требований**| **1. Свойства пакета при регистрации:** <br> 1.1 Тип образца с выбором одного варианта из списка: <br> - Минерал (пока одно значение)<br> 1.2. Вес, заявленный клиентом. Обязательно для заполнения, ввод вручную. <br> 1.3 Вес, измеренный в лаборатории на ювелирных весах. Обязательно для заполнения. <br> 1.4 Система сертификации (с возможностью множественного выбора): <br>- Сертификация по РФ системе <br> - Сертификация по международной системе <br> 1.5. Состав дополнительных работ (с возможностью множественного выбора, работ может не быть): <br> - Съемка <br> - Гравировка с вводом текста гравировки <br> 1.6. Вид сертификационного документа, обязательно для заполнения: <br> - Cертификат соответствия. Выбрано по умолчанию <br> - Экспертное заключение.|
